## A. Claude 실행 프로토콜 (적용하면 이 순서대로)

**STEP 1 — 현황 감지 (먼저 조사, 질문은 그다음).** 각 항목의 §"감지" 절차로 **이미 설치됐는지 / 유사한 것이 있는지** 확인한다:
- claude-hud: `~/.claude/settings.json`의 `statusLine` 키, `~/.claude/plugins/` 아래 claude-hud 존재 여부. **다른 statusLine 구성이 이미 있으면 "유사 구성"으로 분류.**
- svg-design·frontend-design: `~/.claude/skills/`와 `<프로젝트>/.claude/skills/`의 동명 폴더, 그리고 **비슷한 목적의 스킬**(이름·description에 svg/design/frontend/ui 포함) 존재 여부.

**STEP 2 — 체크리스트 확인 요청.** AskUserQuestion으로 묻는다(감지 결과를 각 선택지 설명에 표시):
1. **설치 항목 선택** (multiSelect): §1 claude-hud / §2 svg-design / §3 frontend-design — 각 항목에 현재 상태를 병기한다. 예) "svg-design — 미설치", "claude-hud — ⚠️ 이미 다른 statusLine 사용 중".
2. **이미 설치됐거나 유사 구성이 감지된 항목**은 반드시 처리 방법을 묻는다: **유지(건너뜀) / 재구성(설정만 갱신) / 교체(기존 제거 후 설치)**. 묻지 않고 덮어쓰지 않는다.
3. **스킬 항목(§2·§3)은 스코프**를 묻는다: **글로벌**(`~/.claude/skills/` — 이 PC 전 프로젝트) / **프로젝트**(`.claude/skills/` — 이 repo만, 커밋해 팀 공유 가능). claude-hud는 구조상 **글로벌 전용**이므로 묻지 않고 고지만 한다.
4. **스킬 항목은 호출 방식**을 묻는다:
   - **자동 판단** — Claude가 관련 작업을 감지하면 스스로 로드해 적용 (frontmatter 기본값)
   - **수동 호출 전용** — `/svg-design`처럼 사용자가 부를 때만 (frontmatter에 `disable-model-invocation: true`)

**STEP 3 — 체크된 항목만 설치.** 각 항목의 §"설치" 절차 실행. 선택 안 된 항목은 손대지 않는다.

**STEP 4 — 검증·보고.** 각 항목의 §"검증"을 실행해 **증거(출력)로 보고**. 프로젝트 스코프로 설치한 파일이 있으면 커밋 안내(글로벌 설치는 커밋 대상 아님).

> 원칙: 승인 없이 기존 구성을 덮어쓰지 않는다. `settings.json` 수정은 **해당 키만 병합**(통째 교체 금지). 시크릿은 읽지도 커밋하지도 않는다.

## 1. claude-hud — CLI 사용량 실시간 HUD

**무엇**: 입력창 아래에 상태줄(statusLine)을 그려주는 플러그인. 컨텍스트 사용량 바(초록→노랑→빨강), 구독 사용량(rate limit) 소모율, git 브랜치, 도구/에이전트 활동, todo 진행률 표시.
**스코프**: 글로벌 전용(`~/.claude/plugins/` + user `settings.json`의 `statusLine`) — 이 PC의 모든 세션에 적용.
**요구사항**: Claude Code v1.0.80+, Node.js 18+(Windows 포함. 없으면 `winget install OpenJS.NodeJS.LTS`).

### 감지
- `~/.claude/settings.json`에 `statusLine` 키가 있는가? 있다면 그 값이 claude-hud인가, 다른 도구인가(다른 도구면 **교체 여부를 반드시 질문**).
- `~/.claude/plugins/` 아래 `claude-hud` 폴더(또는 `/plugin` 목록)가 있는가? 있으면 설치 대신 **재구성**(`/claude-hud:configure`)을 제안.

### 설치 (세션 안에서 순서대로)
```text
/plugin marketplace add jarrodwatts/claude-hud
/plugin install claude-hud
/reload-plugins
/claude-hud:setup        ← 프리셋(Full/Essential/Minimal) 선택 + statusLine 자동 구성
```
- 설정 파일: `~/.claude/plugins/claude-hud/config.json` (언어·표시 항목·색상). 이후 조정은 `/claude-hud:configure`.
- 출처: `github.com/jarrodwatts/claude-hud`

### 검증
새 세션(또는 `/reload-plugins` 후)에서 입력창 아래 HUD 줄이 보이는지 확인. `~/.claude/settings.json`에 `statusLine` 키가 생겼는지 확인해 보고.

### 제거
`/plugin uninstall claude-hud` + `settings.json`의 `statusLine` 키 제거(다른 키는 손대지 않음).

## 2. svg-design — SVG 필터 디자인 스킬

**무엇**: 웹 UI의 시각 효과(손그림 떨림·Gooey·Liquid Glass·네온 타이포)를 **SVG 필터로 정확하게** 구현하게 하는 스킬. AI가 흔히 틀리는 지점(ID 중복·Safari 클리핑·성능·접근성·XSS)을 규칙으로 강제한다.
**적용 시점**: 사용자가 고른 호출 방식에 따라 — 자동 판단(관련 작업 감지 시) 또는 `/svg-design` 수동 호출.

### 감지
`~/.claude/skills/svg-design/`·`.claude/skills/svg-design/` 존재 여부 + 유사 스킬(svg·filter·effect 계열). 있으면 유지/교체 질문.

### 설치
선택된 스코프의 `skills/svg-design/SKILL.md`로 아래 내용을 생성한다. (수동 호출 전용을 골랐으면 frontmatter에 `disable-model-invocation: true` 한 줄 추가.)

```markdown
---
name: svg-design
description: 웹 프론트엔드에서 SVG 필터 기반 시각 효과(손그림 떨림·Gooey/Metaball·Liquid Glass·네온 텍스트 등)를 구현·수정할 때 사용. 인라인 SVG 필터 코드 작성, React/Next.js 컴포넌트화, Safari 호환·성능·접근성 문제 해결에 적용.
---
# SVG 필터 구현 규칙 (Staff-Level Frontend 기준)

너는 React/Next.js·SVG 그래픽 전문 아키텍트다. 아래 규칙을 예외 없이 지켜 결함 없고 접근성 있는 고성능 인라인 SVG 필터를 생성한다.

## 필수 제약 (위반 금지)
1. **SSR/Hydration**: 컴포넌트가 반복 렌더링될 수 있으면 `<filter id>`를 하드코딩하지 말 것. React 18 `useId()`로 고유 ID 생성(`useId().replace(/:/g,'')`) 후 인라인 style로 `filter: url(#${id})` 연결. `Math.random()` ID는 Hydration 에러를 유발하므로 금지.
2. **Safari(WebKit) 클리핑 버그**: `feGaussianBlur`·`feDisplacementMap` 적용 요소는 효과가 경계 밖으로 나가면 잘린다(`overflow: visible`로 해결 안 됨). 래퍼 div에 여유 padding 또는 `transform: translateZ(0)` 부여. 필터에 `x="-10%" width="120%"` 여백이나 `filterUnits="userSpaceOnUse"`를 명시.
3. **성능**: `feTurbulence`의 `numOctaves`는 3 이하(모바일 1 권장). 매 프레임 React state로 SVG 속성을 바꾸지 말 것 — GPU 캐시가 무효화된다. 애니메이션은 CSS transform 또는 SMIL(`<animate>`)로 컴포지터 스레드에서. 확대/축소 왜곡 방지엔 `primitiveUnits="userSpaceOnUse"`.
4. **접근성**: 텍스트에 필터를 걸 때 `<path>` 아웃라인화 금지 — DOM 텍스트(`<h1>` 등)를 유지한 채 CSS `filter: url()`만 적용(선택·복사·스크린리더 보존). 시각 전용 `<svg>` 정의부엔 `aria-hidden="true"`, width/height 0.
5. **보안**: 외부 입력이 SVG에 들어오면 DOMPurify(`USE_PROFILES: { svg: true, svgFilters: true }`) 필수. 사용자 값을 `href`·`in2`에 비이스케이프로 넣지 말 것(픽셀 탈취·XSS 벡터).

## 핵심 레시피
- **손그림 떨림(Squiggle)**: `feTurbulence`(baseFrequency 0.02, numOctaves≤3) → `feDisplacementMap`(scale 5, R/G 채널) + `<animate attributeName="seed" values="0;1;2;3;4" dur="0.4s" calcMode="discrete" repeatCount="indefinite">` — JS 개입 없이 렌더 파이프라인 안에서 애니메이션.
- **Gooey/Metaball**: 그룹 컨테이너에 `feGaussianBlur(stdDeviation 6~10)` → `feColorMatrix`(alpha 행 `0 0 0 18 -7` 류로 대비 극대화) → `feComposite(operator="atop")`. 알파를 0/1로 양극화해 달라붙는 점성 연출. `colorInterpolationFilters="sRGB"` 명시.
- **Liquid Glass**: 렌즈형 변위 맵 이미지를 `feImage`로 로드 → `feDisplacementMap`(R=x축, G=y축 굴절) + `feComponentTransfer`로 하이라이트. CSS는 `backdrop-filter: url('#id') blur(12px)` — Chromium 우선, Safari/Firefox엔 순수 CSS 블러 폴백 필수.
- **네온/마스킹 타이포**: 숨긴 `<svg aria-hidden>`에 `feGaussianBlur → feMerge(blur + SourceGraphic)` 필터 정의, 실제 `<h1>`에 `filter: url()` 적용.

## 출력 지침
Gooey·Squiggle·Liquid Glass 등 요청 시 Tailwind 클래스를 쓰는 TypeScript React(.tsx) 컴포넌트 코드를 바로 출력하고, 적용된 수학 원리(feColorMatrix 임계값 클램핑, 변위 공식 등)를 2~3문장으로 덧붙인다. 프레임워크가 React가 아니면 그 스택 관용구로 변환하되 위 제약은 동일하게 지킨다.
```

### 검증
`/` 자동완성(또는 `/context`)에 svg-design이 뜨는지 확인. 스킬을 호출해 "Gooey 버튼 컴포넌트"를 생성시켜 **useId 사용·Safari 대응·aria-hidden**이 코드에 들어갔는지 확인해 보고.

### 제거
해당 스코프의 `skills/svg-design/` 폴더 삭제.

## 3. frontend-design — AI 티 안 나는 프론트엔드 디자인 스킬

**무엇**: Anthropic 공식 스킬. 흔한 "AI가 만든 UI" 룩(뻔한 폰트·보라 그라데이션·과한 애니메이션)을 피하고, 주제·톤에 맞는 **의도적 디자인**(타이포·컬러·모션·공간 구성)의 프로덕션급 HTML/CSS/JS·React·Vue 코드를 만들게 한다.
**적용 시점**: 사용자가 고른 호출 방식에 따라 — 자동 판단 또는 수동 호출.

### 감지
`skills/frontend-design/` 존재 여부(양 스코프) + 유사 스킬. **§2 svg-design과는 역할이 다름**(frontend-design=디자인 방향·전체 UI, svg-design=SVG 필터 효과 구현) — 둘 다 설치해도 충돌하지 않음을 사용자에게 고지.

### 설치
- **프로젝트 스코프**: 루트에서 `npx skills add https://github.com/anthropics/skills --skill frontend-design` 실행(`.claude/skills/frontend-design/` 생성).
- **글로벌 스코프**: 위 명령이 프로젝트에 설치하므로, 생성된 `frontend-design/` 폴더를 `~/.claude/skills/`로 이동(또는 `github.com/anthropics/skills`의 `frontend-design` 폴더를 받아 복사).
- npx 실행이 실패하면(네트워크/권한) 저장소에서 스킬 폴더를 직접 내려받아 복사하는 폴백을 쓴다.
- 수동 호출 전용 선택 시 SKILL.md frontmatter에 `disable-model-invocation: true` 추가.
- 출처: `skills.sh/anthropics/skills/frontend-design` · `github.com/anthropics/skills`

### 검증
`/` 자동완성에 frontend-design이 뜨는지 + SKILL.md frontmatter가 온전한지 확인해 보고. (Claude Code에 내장 frontend-design 플러그인 제안이 뜨는 경우가 있는데, 이 스킬을 설치했으면 중복 설치하지 않는다.)

### 제거
해당 스코프의 `skills/frontend-design/` 폴더 삭제.
